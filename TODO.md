# Eva
Работаем в последней версии Android Studio, это, кажется "Panda 2"

## Как работать с кодом
Все сделано в архитектуре MVVM согласно методичке на задания 7-8 из 3 модуля
Основные рабочие объекты хранятся как дата-классы в domain/model

ActivitiesRepository определяет интерфейс возможных действий над объектами.
Реализация репозитория сделана в ActivitiesRepositoryImpl

На View слое используется обёртка для использования этой Impl, все действия
выполняются через UseCase классы

Чтобы добавить новое действие в систему, надо расширить базовый data класс (если необходимо),
потом обновить его репозиторий, расширив интерфейс, потом имплементировать новую функцию в
Impl, потом подключить использование этой функции как UseCase. Затем можно будет использовать
всё это дело как обычно

Для референсов см. примеры, как это реализовано

## Как работает время и этапы
Используем время Instant из Clock.System.now(). Все этапы в трекере идут сразу друг за другом, в них
не может быть пустоты. Этапы не могут быть выключены, только переключены. Конец одного этапа - это
начало нового этапа. При удалении текущего этапа, предыдущий занимает всё его время.

Для этапов надо реализовать: переключение, удаление последнего этапа, добавление заметки к этапу,
изменение времени конца этапа.
